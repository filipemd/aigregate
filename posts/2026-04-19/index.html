Apple Silicon’s Zero-Copy GPU Breakthrough#

A new implementation enables WebAssembly modules to share physical memory directly with the GPU on Apple Silicon, removing the data-copying bottleneck. This allows for near-zero overhead AI inference and the ability to serialize and move ‘stateful’ AI conversation caches (KV caches) between machines or models.

C++26: Memory Safety and Reflection#

The C++26 standard draft is complete, bringing significant memory safety improvements that can be applied to existing code via recompilation. It also introduces reflection, allowing the language to describe itself to simplify metaprogramming without runtime overhead.

  • https://www.infoq.com/news/2026/04/cpp-26-reflection-safety-async/

The ‘Zero-Bloat’ Postgres Queue#

PgQue is a pure SQL implementation of the battle-tested PgQ architecture, designed to provide a durable event stream inside Postgres without the performance decay and index bloat typical of standard in-database queues.

Linux 7.0 Scheduler Performance Risks#

The Linux 7.0 kernel’s new default lazy-preemption mode caused a 50% performance drop in some PostgreSQL benchmarks. The issue is triggered when huge pages are disabled, highlighting a critical dependency between scheduler configuration and memory management for high-load databases.

  • https://lwn.net/SubscriberLink/1067029/d4520f2073e2d1bf/

Hardware Hacking: Decapsulating the ME2#

A researcher successfully reverse-engineered a niche 2008 handheld device by physically removing the epoxy ‘glob-top’ from the microcontroller to identify the silicon die. This led to the discovery of the μ’nSP instruction set and the ability to modify the device’s internal flash memory via custom USB commands.

  • https://github.com/coremaze/ME2-Writeup/blob/master/README.md
